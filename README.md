# erikacfreeman.com

Source for [erikacfreeman.com](https://erikacfreeman.com). Built with [Quarto](https://quarto.org), deployed to GitHub Pages from the `gh-pages` branch.

Live (staged) preview: <https://erikafreeman.github.io/website/>. The custom domain is not pointed here yet — see **Deploy** below.

## Day-to-day

Most updates are one of three things:

```bash
# Preview locally (auto-rebuilds on save)
quarto preview

# Build once
quarto render

# Publish the site (renders + pushes to the gh-pages branch)
quarto publish gh-pages

# Back up the source too
git add . && git commit -m "update" && git push
```

Or just tell Claude what to update and it'll edit the right file, commit, and push.

## Structure

| File                  | What it is                                            |
|-----------------------|-------------------------------------------------------|
| `data/publications.yml` | **Single source of truth for papers** (edit here)   |
| `data/cv.yml`         | **CV content** — everything except papers (edit here) |
| `scripts/build.py`    | Generates `publications.qmd` + `cv.qmd` from the data  |
| `_quarto.yml`         | Site config: theme, includes, `pre-render` hook        |
| `custom.scss`         | All design tokens (palette, type, layout)             |
| `styles.css`          | Google Fonts import                                   |
| `_header.html`        | Masthead injected on every page                       |
| `_footer.html`        | Colophon injected on every page                       |
| `index.qmd`           | Hero + bio                                            |
| `projects.qmd`        | Project-card grid                                     |
| `impact.qmd`          | Research impact stat blocks                           |
| `publications.qmd`    | *Generated* — do not edit (comes from the YAML)        |
| `cv.qmd`              | *Generated* — CV page (HTML) + downloadable PDF        |
| `contact.qmd`         | IGB address + links                                   |
| `assets/`             | Images (portrait, project figures, favicon)           |
| `requirements.txt`    | Python dep (pyyaml) the build script needs            |

### How the publications + CV stay in sync

`data/publications.yml` is the **one place** papers live. On every build (locally
via `quarto render` or `quarto publish gh-pages`), `scripts/build.py` runs and
regenerates both the website Publications page **and** the CV (page + PDF) from
it. Add one paper to the YAML, redeploy, and the site page, the CV page, and the
downloadable `cv.pdf` all update together.

## Updating content

- **New / updated paper** → edit `data/publications.yml` (updates site + CV + PDF)
- **CV change** (position, grant, talk, award…) → edit `data/cv.yml`
- **New project** → add a `.project-card` block in `projects.qmd`
- **Bio change** → edit `index.qmd` (the `.bio-grid` block)
- **Contact change** → edit `contact.qmd`
- **Visual tweak** → edit `custom.scss` (one place for everything)

> `publications.qmd` and `cv.qmd` are generated by `scripts/build.py`. Don't edit
> them by hand — your changes get overwritten on the next build. Edit the YAML.

## Deploy

Hosted on GitHub Pages from the **`gh-pages` branch** of
`github.com/erikafreeman/website`, currently at
<https://erikafreeman.github.io/website/>. Redeploy after changes with:

```bash
quarto publish gh-pages   # renders, then pushes the built site to gh-pages
```

### Point erikacfreeman.com here (go-live)

1. Recreate a `CNAME` file at the repo root with one line: `erikacfreeman.com`,
   then run `quarto publish gh-pages` so the custom domain ships in the build.
2. Repo → Settings → Pages → Custom domain: `erikacfreeman.com`.
3. At your DNS host, replace the Canva records with GitHub Pages:
   - apex `erikacfreeman.com` → A records `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
   - `www` → CNAME `erikafreeman.github.io`
4. After DNS propagates, tick "Enforce HTTPS" in Pages settings.

### Optional: switch to push-to-deploy CI

Restore `.github/workflows/publish.yml` (it's in git history), grant the gh login
the `workflow` scope (`gh auth refresh -h github.com -s workflow`), set
Settings → Pages → Source: GitHub Actions, and every `git push` renders + deploys.
